    22-03-2024 Scenario Question:
    You have been given a task to automate a process in Salesforce. Upon creating a new account, the system should automatically generate two contacts. Additionally, three opportunities need to be created with specific names: Opp-A1-1, Opp-A1-2, and Opp-A1-3. The opportunities have three status options: Initial, In Progress, and Final. "In Progress" represents 75% completion, and "Final" indicates 100% completion.

    Your objective is to implement a validation rule that triggers when the status of an opportunity is changed from "In Progress" to "Initial." The validation rule should display an error message to prevent this transition from occurring prematurely.

    Outline the steps you would take to create and implement this validation rule, ensuring that it aligns with the specified criteria and error messaging requirements.
    22-03-2024
    • Do's
      1. Design principle - name is prefixed with account name
      2. Call class handler inside a trigger
      3. Always create a method for re-occuring code
      4. Write as minimal apex as possible and use already available and in-built features
      5. Can write comments to increase readability
      6. Use Validation Rules in Object Manager
    • Don't's
      1. Don't create custom objects and custom fields as they come with a cost
      2. Don't hard code probability (contributes to deal value)
